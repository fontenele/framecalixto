            1. Breve descrição
          2. Fluxo de Eventos
            1. Fluxo básico
                O ator deverá acessar a tela - Pesquisa pelo menu do sistema.
              1. A tela - Pesquisa:
                Esta tela tem a funcão de pesquisa e navegação nos registros do cadastro e é definida com os seguintes componentes e opções:
                1. A opção "Novo Registro" para apresentar a tela de inclusão de um novo registro.
                2. A opção "Pesquisar" para efetuar a pesquisa segundo o preenchimento dos campos da pesquisa.

                3. Também deverá apresentar uma listagem para a apresentação dos dados de
                    Em cada item da listagem deverá possuir os links com as ações:
                  1. "Excluir" para acionar a exclusão do registro referente a linha da listagem.
                  2. "Editar" para apresentar a tela de edição do registro referente a linha da listagem.
              2. A tela - Edição:
                Esta tela tem a função de inclusão de um novo registro e a edição de um registro existente, é composta com os seguintes componentes e opções:
                1. A opção "Gravar" para executar a gravação do registro no banco de dados do sistema.
                2. A opção "Excluir" para executar a exclusão do registro no banco de dados do sistema.
                3. A opção "Tela de Pesquisa" para redirecionar o cadastro para a tela: - Pesquisa
          3. Fluxo alternativos
            1. Pesquisa Vazia
              1. Ao se realizar uma pesquisa que não exista registros a serem apresentados, no local da listagem deverá ser apresentado a mensagem “Não foram encontrados registros”.
            2. Exclusão Condicional
              1. Ao clicar em uma opção [Excluir], o sistema deverá solicitar uma confirmação ao ator com a seguinte mensagem
                “Deseja realmente excluir este registro?”.
                Caso a resposta do ator seja afirmativa o sistema deverá realizar a exclusão do registro,
                caso a resposta do ator seja negativa o sistema deverá cancelar a operação.
            3. Gravação sem Dados
            4. Requisitos especiais
              1. Disponibilidade e conectividade do servidor web com a aplicação.
            5. Condições Prévias
              1. O sistema deverá estar configurado e com conetado ao banco de dados.
              2. A tabela "" deverá existir e estar disponível para leitura e gravação.
            6. Condições Posteriores
              1. O cadastro deverá conseguir efetuar as operações de inclusão, pesquisa, alteração e exclusão nos registros da tabela "".
              2. A pesquisa poderá ser efetuada pelos campos disponíveis na tela de pesquisa.
            7. Pontos de Extensão.
              1. Não se aplica.
